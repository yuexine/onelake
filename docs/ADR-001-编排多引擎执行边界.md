# ADR-001：编排多引擎执行边界

> 状态：已接受
> 日期：2026-07-12
> 适用范围：编排模块 V2 M4（B1–B7、G1–G3）

## 1. 背景

M1–M3 的生产执行主线以 Spark 为唯一准入引擎。M4 将增加 `TRINO_SQL`、`PYTHON`、`SHELL` 以及控制/观测类任务；如果直接移除 `RuntimeContractService` 的非 Spark 拦截，会把脚本安全、凭证隔离、资源治理和失败语义同时暴露到生产路径。

本 ADR 固定 M4 的准入边界，使 Java 继续负责校验并生成 runConfig，Dagster 只按已批准的能力执行。

## 2. 决策

1. **Spark 仍是默认生产执行引擎。** `SPARK_SQL`、`PYSPARK` 和由算子模板编译出的 Spark 节点继续走现有 Spark submit、节点回调、重试、日志和资产事件链路。
2. **Trino 是受限 SQL 补充引擎。** `TRINO_SQL` 仅执行结构化 SQL；catalog/schema 必须来自服务端允许列表，不接受客户端连接串、凭证或任意 session property。
3. **Python/Shell 是默认关闭的受限脚本能力。** 只有显式启用脚本执行能力的环境和租户可发布脚本节点；未满足沙箱能力时，编译或触发必须 fail-fast，不得退化为宿主机直接执行。
4. **控制/观测节点不冒充数据引擎。** `BRANCH`、`CONDITION`、`SENSOR`、`WAIT`、`SUB_PIPELINE`、`NOTIFY`、`ASSERTION` 由 Dagster 控制执行器处理，不启动 Spark，也不默认声明资产产出。
5. **Java 生成 runConfig，Dagster 执行。** 任务类型、引擎、资源、超时、允许访问范围和版本锁定由 Java 编译并冻结到发布快照/runConfig；Dagster 不自行推断或放宽权限。

## 3. 能力矩阵

| 能力 | Spark | Trino | Python/Shell | Control/Observe |
| --- | --- | --- | --- | --- |
| 默认状态 | READY | READY_LIMITED | DISABLED/RESTRICTED | READY_BY_TYPE |
| 允许任务 | SPARK_SQL、PYSPARK、Spark 算子 | TRINO_SQL | PYTHON、SHELL | BRANCH 等控制/观测类型 |
| 数据写入 | Iceberg/Hive 既有路径 | 仅允许 catalog/schema 白名单内 SQL | 默认禁止；显式产物目录除外 | 不直接写数据 |
| 网络 | 既有数据面网络 | 仅 Trino 服务 | 默认禁网，按目的地址白名单放行 | 仅批准的内部 API |
| 凭证 | 服务端注入数据面最小凭证 | 服务端 Trino 身份 | 不注入控制面 token/数据库密码 | 内部调用使用专用短期身份 |
| 资源限制 | 现有 Spark profile | 查询超时、结果/日志上限 | CPU、内存、进程数、时间、输出上限 | 轮询间隔与总等待上限 |
| 失败语义 | 节点 FAILED 并短路 | 同 Spark | 超时/越权均 FAILED | 按类型返回 FAILED 或 SKIPPED |

`RuntimeContractService` 必须按此矩阵返回 READY、RESTRICTED、DISABLED 或 BLOCKED，并给出可操作的阻塞原因；不得使用“非 SPARK 一律放行”或“非 SPARK 一律拦截”的单一判断。

## 4. 脚本沙箱硬约束

- 独立容器或等价隔离单元运行，非 root 用户、只读根文件系统、独立临时工作目录。
- 禁止挂载宿主机 Docker socket、源码目录、控制面配置目录和云实例元数据凭证。
- 仅注入业务白名单环境变量；内部 token、数据库密码、Keycloak/MinIO 管理凭证不得进入脚本环境。
- 默认拒绝出网；需要访问的数据面地址按 hostname、端口和用途配置白名单。
- 强制 CPU、内存、进程数、执行时长、stdout/stderr 和产物大小上限；超限后终止整个进程组。
- 日志经过截断和敏感信息清洗后，沿用 M1 task log/callback 通道上传。

本地 Compose 基线使用专用 `onelake_pipeline` Trino 身份和服务端 file access control，
强制仅允许 `iceberg.default/ods/dwd`；显式限定名不能绕过 catalog/schema 边界。
脚本执行同时使用进程级 rlimit 和宿主 `/proc` 进程树聚合监控，按节点统计
CPU、RSS、进程数与工作目录总字节；无法证明该计量能力时 fail closed。
为防止多进程放大节点预算，运行时会按内存/CPU 总额保守计算有效进程上限，
并将总额分配为不可上调的单进程 hard rlimit；工作目录同时限制总字节与文件/目录条目数。
Dagster user-code 容器保持非 root、只读根文件系统、无 capabilities，并使用允许
Bubblewrap namespace 创建但拒绝 ptrace/BPF/跨进程内存访问的定制 seccomp 规则。
能力准入必须同时满足全局 `ONELAKE_SCRIPT_SANDBOX_ENABLED=true` 与
`ONELAKE_SCRIPT_SANDBOX_ALLOWED_TENANTS` 租户 UUID 白名单；两者任一缺失时，编译、发布和触发均拒绝。

任一硬约束在目标部署环境不可证明时，`PYTHON`/`SHELL` 必须保持 DISABLED。

## 5. 版本与复现

- 发布快照必须冻结 `taskType`、engine、operatorRef/operatorVersion、编译后 config 和资源/安全策略引用。
- PROD、CRON、EVENT、BACKFILL 和自动重跑必须继续绑定不可变 `pipeline_version_id`；DEV 试跑可使用草稿但不得覆盖生产指针。
- 算子升级不修改历史快照；重新发布才生成引用新算子版本的生产版本。

## 6. 发布与回退

1. M4 先落能力矩阵和类型分发骨架，再逐引擎启用。
2. Trino 与脚本均使用独立 feature flag/tenant capability，可单独关闭。
3. 关闭补充引擎不会修改已发布快照；新触发应以清晰 BLOCKED 原因拒绝，正在运行的节点按既定超时/取消语义收口。
4. Spark 旧类型和旧发布快照必须作为每一步的回归基线。

## 7. 非目标

- 不在 M4 实现任意第三方执行引擎插件体系。
- 不允许用户上传依赖包并在控制面宿主机执行。
- 不把 Python/Shell 作为绕过 SQL、数据权限或审批的通道。

## 8. 结果

M4 可以引入多引擎，但每种能力都必须在编译期、发布期和触发期受同一能力矩阵约束。安全沙箱不是后续增强项，而是脚本节点进入 READY 的前置条件。
