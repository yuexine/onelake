# 全局任务条当前状态与实施方案

> 修订日期：2026-07-09
>
> 旧文中“没有统一任务表/API”的判断已过期。当前已有 `common.running_task`、`RunningTaskController/Service`、通知中心和前端 `TaskProgressBar`，但跨模块投影完整度仍需继续推进。

## 1. 当前实现

| 层 | 当前代码 |
| --- | --- |
| 数据库 | `common.running_task`、`common.notification` |
| 后端 | `RunningTaskController`、`RunningTaskService`、`NotificationController`、`NotificationService` |
| 前端 | `TaskProgressBar`、`NotificationCenter`、`useGlobalTasks`、`useNotifications` |
| API | `/api/v1/tasks/running`、`/api/v1/tasks/{id}/dismiss`、`/api/v1/notifications` |

## 2. 当前能力

- 展示运行中/近期任务投影。
- 支持 dismiss。
- 通知中心支持列表、标记已读、全部已读。
- 前端有底部全局任务条和通知入口。

## 3. 仍需补齐

| 优先级 | 项目 | 说明 |
| --- | --- | --- |
| P0 | 跨模块投影覆盖 | Integration、Orchestration、SQL Workbench、Analytics Notebook 等长任务应统一写入 running_task |
| P0 | 取消/跳转协议 | 全局任务条需要知道任务来源、详情 URL 和可取消动作 |
| P1 | 状态一致性 | 统一 `QUEUED/RUNNING/SUCCEEDED/FAILED/CANCELLED` 与前端展示 |
| P1 | 通知联动 | 任务完成、失败、审批、告警与通知中心去重合并 |
| P2 | 历史检索 | 运行任务历史和筛选 |

## 4. 文档口径

- “任务条原型存在”不等于所有长任务已接入。
- 记录模块状态时必须写明是否已经写 `common.running_task`。
- mock 任务不得被写成生产闭环。
