# OneLake 技术初始化与当前工程基线

> 修订日期：2026-07-09
>
> 早期“初始化文档”已转为当前工程基线说明。历史计划中“只先做 integration/orchestration/catalog”“Spark/Flink/Kafka 暂不纳入”“dbt-on-Trino 执行 DWD”等表述已被当前代码替代。

## 1. 当前架构

OneLake 由一个模块化单体控制面和一组开源数据面组成：

```text
React/Vite web-console
        |
Spring Boot control plane (onelake-app)
        |
Postgres metadata + Redis + data-plane clients/events
        |
Airbyte / Spark / Dagster / Trino / Iceberg / APISIX / Superset / JupyterHub
```

控制面负责：

- 元数据、配置、任务、审批、审计、事件。
- 调用数据面 API 或 GraphQL。
- 写 Outbox 事件并消费跨模块事件。

数据面负责：

- 采集、执行、查询、存储、可视化运行时。

## 3. 后端工程

`onelake-app/pom.xml` 当前包含 10 个子模块：

```text
module-common
module-integration
module-orchestration
module-catalog
module-modeling
module-quality
module-security
module-dataservice
module-analytics
bootstrap
```

统一分层：

```text
api/          REST Controller、VO
service/      用例编排、事务、事件
domain/       Entity、枚举、领域状态
repository/   Spring Data JPA
client/       数据面适配
dto/          对外/跨层 DTO
event/        事件处理器
config/       模块配置
```

依赖规则：

- 各业务模块依赖 `module-common`。
- 模块之间不直接 import 对方的 `domain/repository/entity`。
- 跨模块协作通过 Outbox 事件或公共投影表完成。

## 4. 数据库

Spring Flyway 配置的 schema 范围：

```text
common,integration,orchestration,catalog,modeling,quality,security,dataservice,dataservice_api,analytics
```

迁移目录为 9 个拥有表的 schema；`dataservice_api` 用作 PostgREST/API 视图 schema。当前迁移约创建 73 张表。

本地默认 `spring.flyway.enabled=false`，使用：

```bash
cd onelake-app
make migrate
```

或测试显式开启 Flyway。

## 5. Outbox 事件

当前事件机制：

1. 业务事务内写 `common.outbox_event`。
2. `OutboxDispatcher` 轮询 `PENDING`，发布到 Redis Stream。
3. `RedisStreamDomainEventConsumer` 使用 consumer group 消费。
4. `common.consumed_event` 做事件幂等。
5. Handler 只处理业务逻辑。

事件名集中在 `com.onelake.common.outbox.DomainEvents`。事件清单维护见 `docs/全局事件契约（Outbox 总线 · 事件清单 · 发布订阅 · 载荷 Schema）.md`。

## 8. DWD 当前主线

当前 DWD 职责拆分：

| 模块 | 职责 |
| --- | --- |
| `module-modeling` | DWD 模型草稿、字段映射、校验、编译、发布 |
| `module-orchestration` | Spark-only pipeline、任务/边、运行、回填、资源 |
| `dagster/definitions.py` | `onelake_pipeline_run` 执行 Spark SQL/PySpark/Quality Gate |
| `module-catalog` | 资产、字段 schema、血缘、影响分析 |
| `module-quality` | 质量规则/结果/告警 |

旧 `modeling.model_run`、`/models/{id}/run`、`onelake_dbt_model_run`、`dagster-dbt` 表述不再作为当前主线。
