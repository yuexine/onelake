# 数据分析与可视化模块当前方案

> 修订日期：2026-07-09
>
> 旧文从“进入 P1 实施”的设计稿口径升级为当前实现说明。`module-analytics` 已存在，涵盖数据集、Dashboard、Superset、Notebook、NL2SQL 与分享链路；外部依赖仍需按验收文档单独验证。

## 1. 当前实现

| 区域 | 当前代码 |
| --- | --- |
| 后端模块 | `module-analytics` |
| Controller | `DatasetController`、`AnalyticsDashboardController`、`SupersetEmbedController`、`ShareController`、`NotebookController`、`NotebookTemplateController`、`Nl2SqlController` |
| Entity | `Dataset`、`Dashboard`、`DashboardPublication`、`Notebook`、`NotebookTemplate`、`NotebookRun`、`QueryLog` |
| Client | Trino、Superset、DataService、JupyterHub、Dagster、OpenAI LLM |
| 前端路由 | `/analytics/dashboards`、`/analytics/datasets`、`/analytics/notebooks`、`/analytics/library`、`/share/screen/:token` |
| 数据面 | Superset、JupyterHub、Dagster Notebook job、Trino query |

## 2. 功能状态

| 能力 | 状态 |
| --- | --- |
| 数据集 CRUD/查询 | 已实现，依赖 Trino/API/Notebook source |
| Dashboard 保存/发布/分享 | 已实现 |
| 公开分享 | 已实现独立路由，需遵守 row_filter 限制 |
| Superset guest token | 已实现，Superset 服务需单独验证 |
| Notebook token/run/artifact | 已实现，JupyterHub/Dagster 需单独验证 |
| Notebook 模板 | 已实现 |
| NL2SQL | 已实现，依赖 OpenAI API key |
| 慢查询/Outbox 事件 | 已实现部分事件常量和服务逻辑 |

## 3. 外部依赖风险

| 依赖 | 风险 |
| --- | --- |
| Superset | Compose 有服务和配置，但 admin/init/bootstrap 需单独验收 |
| JupyterHub | 配置使用 `jupyterhub` OIDC client，`make seed` 当前主要创建 `onelake-app` client，SSO 需复核 |
| Dagster | Notebook job 依赖 user-code 镜像和 papermill |
| OpenAI | NL2SQL 需要 `OPENAI_API_KEY` |
| Trino | 数据集查询依赖表存在、权限和脱敏策略 |

## 4. 验收入口

详细验收见 `docs/ANALYTICS_VERIFICATION.md`。验收结果必须区分：

- 代码构建/单测。
- Compose 服务探活。
- 外部依赖实际可用。
- 业务链路可点通。
