## 1. 背景与定位

### 1.1 现状缺口

平台当前 8 个业务模块覆盖「集成 → 湖仓建模 → 编排 → 质量 → 目录血缘 → 安全 → 数据服务」全链路，但**数据生产出来之后的「最后一公里」缺失**：

- 没有给数据分析师/科学家用的**交互式分析环境**——目前只有 `lakehouse/SqlWorkbench`（纯 SQL），无法跑 Python、做特征工程、训练模型、画探索性图表。
- 没有**面向业务的可视化产出物**——`monitor/Overview` 是写死的运营大盘，无法让用户基于自己的数据资产**自助拼装大屏并发布**。
- 数据资产（Iceberg 表 / DWS·ADS 模型 / 数据 API）目前只能被「看见」（catalog）和「取走」（dataservice），无法被**就地分析与展示**。

### 1.2 模块定位

> **数据分析与可视化模块 = 平台的「消费与洞察层」**：把已治理的数据资产，通过①交互式数据研究（Notebook + Python/算法）和②自助可视化大屏（拖拽编排 + 发布），转化为可探索、可展示、可分享的洞察成果。

两个一级能力：

| 子能力 | 一句话 | 核心载体 | 主要用户 |
|--------|--------|----------|----------|
| **A. 数据研究工作台** | 写 Python/PySpark/SQL 做探索分析、特征工程、算法建模 | Notebook（Jupyter） | DE / 数据科学家 / 分析师 |
| **B. 可视化大屏** | 拖拽内嵌组件、绑定数据资产、自定义大屏并发布 | Dashboard（自研编排器 + ECharts） + 嵌入 Superset | DE / 分析师 / 业务消费者 |

二者通过统一的 **「数据集（Dataset）」抽象**衔接：Notebook 产出可注册为数据集；大屏组件消费数据集。

## 2. 设计目标

### 2.1 North Star 能力

1. **资产直连分析**：任意 Iceberg 表 / 模型 / 数据 API，一键进入 Notebook 或加入大屏，无需重新建连接。
2. **自定义脚本与算法**：Notebook 支持 Python / PySpark / SQL(Trino)，预装 pandas / numpy / scikit-learn / statsmodels / Prophet / plotly 等常用算法库，并提供「算法模板」一键生成（聚类、回归、时序预测、相关性分析等）。
3. **交互 + 可调度**：Notebook 既能交互执行（Jupyter kernel），也能参数化后**调度执行**（papermill + Dagster），产出落表/落图，纳入血缘。
4. **丰富的内嵌可视化组件**：优先集成开源图表库，组件库覆盖折线/柱/饼/散点/雷达/漏斗/热力/地图/词云/水球/指标卡/翻牌器/排名榜/表格/装饰边框等 30+ 组件。
5. **自助大屏编排**：拖拽布局（自由网格）、组件绑定数据集 + 字段映射、样式/主题/交互联动配置、自适应分辨率、暗色大屏主题。
6. **发布与分享**：大屏发布生成只读访问链接（带 token、过期、租户隔离），可嵌入第三方、全屏轮播投屏。
7. **治理一致性**：密级随资产沿用（`ClassificationBadge`）、脱敏策略对查询生效、所有动作审计、消费关系回写血缘（大屏/Notebook 作为资产下游）。
8. **架构合规**：独立 schema、零跨 schema 直写、Outbox 产出事件、数据面「驱动不重造」。

### 2.2 明确边界（不做 / 后做）

- 不自研 BI 查询引擎与图表渲染内核——查询走 Trino，渲染用 ECharts/Superset。
- 不自研 Notebook 内核与编辑器——直接嵌入 JupyterLab。
- 本轮 Notebook 调度仅支持 papermill 参数化批跑，不做 Notebook 内的可视化 DAG（Elyra 列为 P5 可选增强）。
- 实时流式大屏（秒级刷新）本轮降级为「定时轮询（≥5s）」，真正的 WebSocket 推流列为后续。
- 大屏组件联动本轮支持「全局筛选器 → 组件」单向联动，组件间钻取列为后续。
- **Superset 嵌入前提**：被嵌入的 Superset 仪表盘直连的数据源表（Iceberg dwd/dws）必须暴露 `tenant_id` 列，否则 RLS 名存实亡；缺列数据源不允许签发 guest token。
- **公开分享边界**：`/share/screen/:token` 走 guest 通道（无 `TenantContext`），带 `row_filter` 的数据集**不得用于公开分享**——前端发布流程对这类数据集硬拦截，仅放行"已聚合 / 无行级过滤"的数据集。

## 3. 竞品 / 开源调研与选型结论

### 3.1 可视化大屏（B 能力）

| 方案 | 形态 | 借鉴 / 取舍 |
|------|------|------------|
| **Apache Superset**（已部署 :8088） | 企业级 BI，40+ 图表、SQL Lab、RLS、`@superset-ui/embedded-sdk` 嵌入 | **采用**：作为「治理型 BI 面板」直接嵌入；复杂分析图表、跨库探索交给它，不重造 |
| **Apache ECharts** | JS 图表库，图表类型最丰富、大屏主题成熟 | **采用**：自研大屏的渲染内核（`echarts-for-react`） |
| **AntV G2Plot / L7** | 统计图表 / 地理可视化（GIS、热力地图） | **采用（补充）**：地图类、特殊统计图用 L7/G2Plot 补齐 ECharts |
| **react-grid-layout** | React 拖拽/缩放栅格布局 | **采用**：大屏画布的拖拽编排内核 |
| GoView（Vue3） | 开源大屏低代码平台，组件 schema 设计优秀 | **借鉴不直用**：前端是 React，无法直接复用 Vue 组件；**借鉴其组件 JSON Schema、配置面板分区（属性/数据/动画）设计** |
| DataRoom / DataEase / Grafana | 独立大屏/监控系统 | **不内置**：避免再引入一套带库的重系统；Grafana 运维面板可按 iframe 选择性嵌入（可选） |

**结论**：可视化采用「**双轨**」——① 治理型分析直接**嵌入 Superset**（零开发、能力最全）；② 自定义大屏用 **react-grid-layout + ECharts + AntV** 自研编排器（满足「自定义大屏 + 投屏 + 品牌主题」诉求，组件 schema 借鉴 GoView）。两者都以平台「数据集」为数据源，统一治理。

### 3.2 数据研究（A 能力）

| 方案 | 角色 | 取舍 |
|------|------|------|
| **JupyterLab / JupyterHub** | 多用户交互式 Notebook IDE | **采用**：JupyterHub 按用户拉起隔离 kernel；前端 iframe 嵌入 JupyterLab |
| **papermill** | 参数化执行 .ipynb | **采用**：调度态把 Notebook 当作可参数化批处理脚本 |
| **nbconvert** | Notebook → HTML/PDF | **采用**：产出报告归档、分享 |
| **Spark (PySpark)**（已部署） | 大数据计算内核 | **采用**：重计算走 PySpark kernel；轻量分析走本地 pandas + Trino |
| **Trino**（已部署） | 交互式 SQL on Iceberg | **采用**：`trino` python client / SQLAlchemy 让 Notebook 直查湖仓 |
| **Dagster**（已部署） | 调度编排 | **采用**：新增 `onelake_notebook_run` op 调度 papermill |
| 算法库 | pandas / numpy / scikit-learn / statsmodels / prophet / plotly | **预装进镜像** |
| Elyra | JupyterLab 内可视化 pipeline | **P5 可选** |

**结论**：分析能力 = JupyterHub（交互） + papermill/Dagster（调度），计算下沉 Spark/Trino，**全部复用已部署组件，仅新增 1 个 JupyterHub 服务**。

### 3.3 引入的第三方组件清单（汇总）

后端 / 数据面：

| 组件 | 版本基线 | 用途 | 部署方式 |
|------|---------|------|----------|

| 算法库 | pandas/numpy/scikit-learn/statsmodels/prophet/plotly | Python 分析 | jupyter 镜像 |

前端（`web-console`，pnpm 新增）：

|----|------|

## 4. 信息架构与前端原型

### 4.1 IA（菜单与路由）

在 `App.tsx` 的 `NAV` 与 `routes.tsx` 中新增一级菜单「数据分析」`/analytics`（建议置于「数据服务 DaaS」之后）：

```
数据分析 (Analytics)                      /analytics
├── 大屏中心        Dashboards            /analytics/dashboards          # 大屏列表（我的/共享/已发布）
│   └── 大屏设计器   Screen Designer       /analytics/dashboards/:id      # 拖拽编排（核心）
│   └── 大屏预览     Preview/Publish       /analytics/dashboards/:id/view # 只读发布态/投屏
├── 分析工作台      Notebook Workbench    /analytics/notebooks           # Notebook 列表
│   └── Notebook    JupyterLab(嵌入)      /analytics/notebooks/:id       # 交互编辑
├── 数据集          Datasets              /analytics/datasets            # 数据集管理（资产绑定 + SQL）
└── 组件 / 模板库    Library               /analytics/library            # 可视化组件库 + 算法模板库
```

> 公开分享链接走独立无鉴权布局路由 `/share/screen/:token`（不挂在 `<App />` layout 下，仅渲染只读大屏 + guest 数据通道）。

#### 自研大屏 vs Superset 嵌入分流决策

「大屏中心」是统一入口；用户在创建时由向导引导选型，避免"两套入口不知选谁"：

| 用户诉求 | 推荐方案 | 入口 |
|----------|----------|------|
| 指标卡 + 翻牌器 + 排名榜 + 装饰边框 + 投屏大屏（暗黑主题、自由布局） | **自研大屏**（react-grid-layout + ECharts） | 「新建大屏」 |
| 复杂多维分析、交叉过滤、SQL Lab、跨库探索 | **嵌入 Superset**（不重造 BI） | 「新建大屏」→ 拖 `superset` 组件 |
| 已有 Superset 仪表盘想纳入平台治理 | 嵌入 Superset + Outbox 回写血缘 | 「导入 Superset 面板」 |
| 简单单图（一张折线/饼图） | 二选一均可，默认自研（轻量） | 「新建大屏」→ 拖一个组件 |

> Inspector 三 Tab（属性/数据/交互）始终一致，用户对底层渲染内核无感。

### 4.2 大屏设计器原型（核心交互，对应原型 §8.x 风格）

```
┌─ PageHeader: 大屏 [华东销售作战屏] [草稿] ──── 预览 · 保存 · 发布 ──── 1920×1080 ▾ · 主题:暗黑 ▾ ─┐
├──────────────┬──────────────────────────────────────────────┬──────────────────────────────┤
│ 组件库 (palette)│                 画布 (react-grid-layout)         │  检查器 (Inspector)            │
│ ▸ 图表         │   ┌──────────┐ ┌───────────────┐              │  [属性] [数据] [交互/动画]      │
│  · 折线/柱/饼   │   │ 指标卡:GMV │ │   折线图        │              │  ── 数据 Tab ──               │
│  · 散点/雷达    │   │ ¥12.3M ↑6% │ │  (echarts)     │              │  数据集: ads_order_gmv_daily ▾ │
│  · 漏斗/热力    │   └──────────┘ └───────────────┘              │  [+ 新建数据集]                │
│ ▸ 地图(L7)     │   ┌────────────────────────────┐              │  维度(x): stat_date            │
│ ▸ 指标/翻牌器   │   │   中国地图(省份销售热力)        │              │  指标(y): gmv [sum]            │

│ ▸ 文本/媒体     │   ┌────────────┐ ┌────────────┐              │  刷新: 30s  缓存:5min          │
│ ▸ 装饰/边框     │   │ Superset面板 │ │  滚动排名榜  │              │  密级: ● L2 内部               │

│ ─────────────  │                                              │                              │
│ 拖拽入画布 →    │   (栅格吸附 · 缩放 · 多选对齐 · 图层 z-index)     │                              │
└──────────────┴──────────────────────────────────────────────┴──────────────────────────────┘
```

三段式与编排「统一编辑器」一致（palette + canvas + inspector），降低用户学习成本。Inspector 三个 Tab：**属性**（样式/标题/配色）、**数据**（数据集 + 字段映射 + 筛选 + 刷新）、**交互/动画**（联动、跳转、轮播）。

### 4.3 分析工作台原型

```
┌─ Notebook [客户分群_RFM] · kernel: PySpark(运行中) ── 运行 · 保存 · 参数化 · 调度 · 导出HTML ─┐

│ ▸ Iceberg 表   │   [2] from sklearn.cluster import KMeans ...                               │
│  · dwd_user…   │       (交互输出图表 / 表格)                                                 │
│ ▸ 算法模板      │   [3] dataset.publish(df, "ads_user_rfm_seg")  # 产出注册为资产/数据集       │
│  · KMeans 分群  │                                                                           │
│  · Prophet 预测 │                                                                           │
│  · 相关性分析    │                                                                           │

└──────────────┴───────────────────────────────────────────────────────────────────────────┘
```

左侧「资产/数据集侧栏」由平台注入（postMessage 与 iframe 通信），点击「插入」把读取该资产的代码片段写入当前 cell。

### 4.4 数据集管理 / 组件库

- **数据集**：列表 + 新建向导（来源：① 选 Iceberg 资产自动生成 `SELECT *`；② 自定义 Trino SQL；③ 绑定已发布数据 API；④ Notebook 产出）。配置字段 schema、缓存 TTL、行级过滤、密级（继承资产）。
- **组件库**：可视化组件目录卡片（预览缩略图 + 默认配置），支持收藏；**算法模板库**：参数化 Notebook 模板，一键「用此模板新建 Notebook」。

### 5.2 与既有模块的协作（全部走句柄 + Outbox，零跨 schema）

- **资产接入**：`analytics.dataset` 仅存资产 **FQN**（如 `iceberg.dwd.dwd_user_codex`）+ 密级快照，不读 `catalog` schema。资产选择器走 catalog 既有 REST（前端调用），运行时查询直接打 Trino。
- **脱敏/权限（全部下推到 Trino，不在 Java 层做行后处理）**：`SqlBuilder.compose()` 在拼 SELECT 时，对带脱敏策略的列直接替换为 Trino mask 表达式（如 `try_cast(mask_phone(mobile) AS varchar)`），由 Trino 引擎层强制执行；脱敏策略经 Outbox 从 `module-security` 同步到本模块的只读投影 `MaskingPolicyView`。这样既不违反"零跨 schema 直读"，又避免全量数据过 Java 层内存。密级随资产快照展示，发布大屏对低权限消费者按密级隐藏组件。
- **血缘回写**：大屏发布 / Notebook 产出表时，`OutboxPublisher.publish("analytics.dashboard.published", id, payload)` → `module-catalog` 消费，把大屏登记为资产**下游消费者**（参考 `dataservice` 的 `AssetConsumerBackfillRunner` 模式）。
- **通知/任务条**：Notebook 调度运行、大屏发布等长任务，产出事件被全局任务条 / `monitor` 消费。

### 5.3 安全与治理

- 角色：沿用 `DE / ADMIN / CONSUMER / SEC / OPS`。创建大屏/Notebook/数据集 = `hasRole('DE')`；查看已发布大屏 = `hasRole('CONSUMER')`；公开分享链接 = guest token（无 Keycloak 身份）。
- **Superset guest token** 由后端 `SupersetEmbedService` 用服务账号签发，**带租户 RLS 规则**，禁止前端直接调用 Superset。
- **分享链接**：`DashboardPublication` 持 `shareToken`（随机）、`expireAt`、`tenantId`；公开通道只暴露「已发布快照 + 数据集只读查询」，且查询经密级/脱敏校验。
- **Notebook 隔离**：JupyterHub 每用户独立容器，CPU/内存配额限制；镜像不含平台密钥，连接 Trino/Spark 走短期凭据；审计 `AuditLogger` 记录执行。
- **Superset 数据源前提**：`SupersetEmbedService` 在签发 guest token 前调用 `verifyDatasourceHasTenantColumn(dashboardUuid)`，校验底层数据源表（Iceberg dwd/dws）含 `tenant_id` 列；缺列直接 `BizException(40001, ...)` 拒绝签发，避免 RLS 名存实亡。
- **公开分享 × 行级过滤**：`SharePublishService.publish()` 在签发 `shareToken` 前，遍历 `spec.widgets[].data.datasetId`，若任一数据集的 `row_filter` 非空 → 拒绝发布为 `is_public=true`，仅允许"内部已发布"通道；公开通道仅放行已聚合 / 无行级过滤的数据集。

### 5.4 可观测性

- **查询日志**：`DatasetQueryService.query()` 每次调用写一条 `analytics.query_log`（dataset_id, tenant_id, sql_md5, duration_ms, rows, cache_hit, dashboard_id, created_at）。用途：①慢查询排查；②按租户出日查询量做配额；③作为血缘回写的"实际命中"证据（区别于"声明绑定"）。
- **慢查询告警**：`duration_ms > 5_000` 时发 Outbox `analytics.query.slow` → `monitor` 告警；dashboard 编辑态显示"慢查询"标识，引导用户沉淀为物化模型。
- **指标埋点**：Micrometer 暴露 `analytics_query_duration_seconds{dataset_id=...}`、`analytics_query_cache_hit_total`、`analytics_notebook_run_duration_seconds{status=...}` 到 Prometheus。
- **追踪**：`TrinoQueryClient` / `SupersetClient` / `JupyterHubClient` / `DagsterClient` 共用 `MDCHttpClient` 配置，自动透传 `X-Trace-Id`，可与 `module-common` 既有 traceId 串联。
- **配额**：`analytics.query_log` 按日聚合出每租户的查询量、P95 延迟、缓存命中率，供运维大盘与未来限流决策。

```tsx
interface WidgetDef {
  type: WidgetNode['type'];
  label: string;
  icon: React.ReactNode;
  defaults: Partial<WidgetNode>;
  buildOption: (node: WidgetNode, rows: Record<string, unknown>[]) => unknown; // → ECharts option

export const WIDGET_REGISTRY: Record<string, WidgetDef> = {
  line: {
    type: 'line', label: '折线图', icon: <LineChartOutlined />,
    defaults: { layout: { x: 0, y: 0, w: 8, h: 6, z: 1 } },
    buildOption: (node, rows) => ({

      series: node.data!.measures.map(m => ({
        type: 'line', smooth: true, name: m.field,
        data: rows.map(r => r[m.field]),
      })),
      ...(node.option ?? {}),
    }),
  },
  // bar / pie / scatter / radar / funnel / heatmap / map(L7) / metric / rankList ...
};

// 通用渲染：拉数 → buildOption → 渲染
export function WidgetRenderer({ node }: { node: WidgetNode }) {

  return <ReactECharts option={def.buildOption(node, rows ?? [])} style={{ height: '100%' }} notMerge lazyUpdate />;
}
```

### 7.6 后端 · 数据集查询服务（Trino + Redis 缓存 + 密级校验）

```java
// module-analytics/src/main/java/com/onelake/analytics/service/DatasetQueryService.java
@Service
@RequiredArgsConstructor
public class DatasetQueryService {

    private final DatasetRepository datasetRepo;
    private final TrinoQueryClient trino;               // client/：JDBC 包装 Trino
    private final DataServiceClient dataserviceClient;  // client/：调 dataservice PostgREST（source_type=API）
    private final StringRedisTemplate redis;
    private final MaskingPolicyView maskingView;        // 脱敏策略只读投影（经 Outbox 从 module-security 同步）
    private final QueryLogRepository queryLogRepo;
    private final OutboxPublisher outbox;

    public QueryResult query(UUID datasetId, DataBinding binding, @Nullable UUID dashboardId) {
        UUID tenant = TenantContext.getTenantId();
        Dataset ds = datasetRepo.findByIdAndTenantId(datasetId, tenant)
            .orElseThrow(() -> new BizException(40400, "数据集不存在"));

        long start = System.currentTimeMillis();
        String sqlFingerprint;
        QueryResult result;

        if (ds.getSourceType() == SourceType.API) {
            // ① source_type=API：反向调 dataservice PostgREST，不入 Trino 缓存（dataservice 已有自己的缓存/限流）
            result = dataserviceClient.query(ds.getApiId(), binding);
            sqlFingerprint = "[api:" + ds.getApiId() + "]";

        } else {
            // ② source_type=ASSET / SQL：脱敏**完全下推到 Trino**（mask 表达式注入 SELECT，不在 Java 层做行后处理）
            String sql = SqlBuilder.compose(ds, binding, maskingView.forTenant(tenant));
            sqlFingerprint = sql;
            String cacheKey = "ds:" + tenant + ":" + datasetId + ":" + DigestUtils.md5Hex(sql);

            // single-flight：3 个组件绑同一数据集时只发一次 Trino 查询（Redis SETNX 锁 + double-check）
            result = redisSingleFlight(cacheKey, Duration.ofSeconds(ds.getCacheTtlSec()), () -> {
                List<Map<String, Object>> rows = trino.query(sql);
                return QueryResult.of(rows, ds.getFieldSchema());
            });
        }

        // ③ 异步写查询日志（不影响主路径）—— 慢查询、配额、血缘命中证据
        long durationMs = System.currentTimeMillis() - start;
        queryLogRepo.saveAsync(QueryLog.builder()
            .tenantId(tenant).datasetId(datasetId).dashboardId(dashboardId)
            .sqlMd5(DigestUtils.md5Hex(sqlFingerprint)).durationMs((int) durationMs)
            .rows(result.getRows().size()).cacheHit(result.isCacheHit())
            .createdBy(TenantContext.getUserId()).build());

        if (durationMs > 5_000) {
            outbox.publish("analytics.query.slow", datasetId.toString(),
                Map.of("durationMs", durationMs, "tenantId", tenant, "dashboardId", String.valueOf(dashboardId)));
        }
        return result;
    }
}
```

```java
@Service
@RequiredArgsConstructor
public class SupersetEmbedService {

    private final SupersetClient superset;   // client/：服务账号登录 + 调用 /api/v1/security/guest_token/

    /** 为当前租户用户签发受 RLS 限制的 guest token（绝不暴露服务账号给前端）。 */
    public String guestToken(String dashboardUuid) {
        UUID tenant = TenantContext.getTenantId();
        // 前置校验：被嵌入 dashboard 的所有数据源表必须含 tenant_id 列，否则 RLS 名存实亡
        verifyDatasourceHasTenantColumn(dashboardUuid);
        var rls = List.of(Map.of("clause", "tenant_id = '" + tenant + "'")); // 行级隔离
        var payload = Map.of(
            "user", Map.of("username", TenantContext.getUsername()),
            "resources", List.of(Map.of("type", "dashboard", "id", dashboardUuid)),
            "rls", rls
        );
        return superset.createGuestToken(payload);  // POST /api/v1/security/guest_token/
    }
}
```

```java
    private final NotebookRepository notebookRepo;
    private final NotebookRunRepository runRepo;
    private final DagsterClient dagster;          // 复用既有 GraphQL launchRun 模式
    private final OutboxPublisher outbox;

    @Transactional
    public NotebookRun schedule(UUID notebookId, Map<String, Object> params) {
        Notebook nb = notebookRepo.findById(notebookId).orElseThrow();
        NotebookRun run = new NotebookRun();
        run.setNotebookId(notebookId);
        run.setTenantId(TenantContext.getTenantId());
        run.setParams(JsonUtils.toJson(params));
        run.setStatus("PENDING");
        runRepo.save(run);

        // 驱动 Dagster：onelake_notebook_run op 内部用 papermill 执行 .ipynb（计算落 Spark/Trino）
        String dagsterRunId = dagster.launchRun("onelake_notebook_run", Map.of(
            "notebook_path", nb.getStoragePath(),
            "parameters", params,
            "run_id", run.getId().toString()
        ));
        run.setDagsterRunId(dagsterRunId);

        // 同事务发事件：任务条/通知消费；完成回调由 Dagster → 事件更新状态
        outbox.publish("analytics.notebook.run-submitted", run.getId().toString(),
            Map.of("notebookId", notebookId, "runId", run.getId()));
        return run;
    }
}
```

> **状态回写（单向轮询，避免双向依赖）**：Dagster 端**不主动反调控制面**。控制面侧的 `NotebookRunSyncScheduler`（`@Scheduled(fixedDelayString = "${analytics.notebook.poll-interval:30000}")`）扫描 `status='RUNNING'` 的 `notebook_run`，调 `dagster.runStatus(dagsterRunId)` 拉最新状态并回写——与 `module-integration.AirbyteSyncDriver` 的轮询模式完全一致。超过 30min 仍在 RUNNING 的记录兜底置 `FAILED` 并发 Outbox `analytics.notebook.timeout`，避免永久卡死。

### 7.11 数据面 · Notebook Python SDK（`onelake` 包）

```python
# onelake-app/jupyter/onelake/__init__.py（打入 jupyter 镜像）
import os
import pandas as pd
from trino.dbapi import connect as trino_connect
from onelake._control_plane import ControlPlaneClient  # 调 /api/v1/analytics/*

class DatasetHandle:
    def __init__(self, fqn: str, client: ControlPlaneClient):
        self.fqn = fqn; self._client = client

    def to_pandas(self, limit: int = 1_000_000) -> pd.DataFrame:
        """读路径：直查 Trino（Iceberg on Trino catalog），短期 token 走 header"""
        sql = f"SELECT * FROM {self.fqn} LIMIT {limit}"
        with trino_connect(
            host=os.environ["TRINO_HOST"],
            user=os.environ["ONELAKE_TENANT_ID"],
            http_headers={"X-Onelake-Token": os.environ["ONELAKE_TOKEN"]},
        ) as cur:
            return pd.read_sql(sql, cur)

    def to_spark(self):
        """重计算场景：返回 SparkSession 已注册的视图"""
        from pyspark.sql import SparkSession
        spark = SparkSession.builder.getOrCreate()
        return spark.table(self.fqn.replace(".", "_"))


def dataset(fqn: str) -> DatasetHandle:
    """读：通过 Trino 直查 Iceberg 表（不经控制面 REST，控制面不挡查询热路径）"""
    return DatasetHandle(fqn, ControlPlaneClient.from_env())


def publish(df, name: str, classification: str = "L1", description: str = "") -> str:
    """写：①Spark 写新 Iceberg 表 → ②调控制面 /artifact 注册到 catalog + 建 dataset 记录"""
    from pyspark.sql import SparkSession
    spark = SparkSession.builder.getOrCreate()
    table_fqn = f"iceberg.dwd.{name}"
    df.writeTo(table_fqn).createOrReplace()                       # ① 落 Iceberg
    ControlPlaneClient.from_env().register_artifact(              # ② 控制面发 Outbox 事件
        fqn=table_fqn, classification=classification, description=description,
        produced_by_notebook=os.environ["ONELAKE_NOTEBOOK_ID"])
    return table_fqn
```

> **关键设计**：
> - **读路径** = `dataset(fqn).to_pandas()` / `.to_spark()`，直接打 Trino/Spark，**不经控制面 REST**——控制面不挡查询热路径。
> - **写路径** = `publish(df, name)`，①Spark 落 Iceberg 表 → ②调控制面 REST → 控制面发 Outbox `analytics.notebook.artifact-published` → `module-catalog` 登记资产 + `module-analytics` 建 dataset 记录。
> - **短期凭据**：JupyterHub 的 `pre_spawn_hook` 调控制面 `/api/v1/analytics/notebooks/{id}/issue-token` 拿 1h 过期的 token，写入环境变量；镜像不存长期密钥。
> - **OpenLineage（可选 P5）**：Spark session 装 `openlineage-spark` listener，自动 emit 资产级 lineage 事件到 Marquez/OM，省掉手写 Outbox 的血缘回写。

## 10. 风险与对策

| 风险 | 对策 |
|------|------|
| Superset guest token 泄露 / 越权 | token 仅后端服务账号签发、带租户 RLS、短过期；前端禁止直连 Superset API |
| Notebook 执行任意 Python（安全） | JupyterHub 每用户隔离容器 + 资源配额；镜像不含密钥；连数据面用短期凭据；全程审计 |
| 大屏直查 Trino 压力 / 慢查询 | Dataset 强制缓存 TTL + 行数上限；轮询≥5s；重查询建议沉淀为物化模型（走编排流水线） |
| 跨 schema 直读冲动 | 资产一律 FQN 句柄 + Outbox 投影；查询打 Trino 而非读 catalog 库 |
| 自研大屏组件维护成本 | 组件 schema 化 + 注册表插件化；优先 ECharts 内置类型，特殊图用 AntV，避免逐个手写 |
| 前端依赖膨胀（ECharts/AntV/L7 体积） | 路由级 `lazy` 懒加载 `/analytics/*`；图表库按需引入 |
| 同数据集多组件并发查询打爆 Trino | Redis SETNX single-flight + 双重 check + 强制 TTL；`DatasetQueryService` 是查询的唯一入口 |
| 公开分享 token × row_filter 数据泄露 | `SharePublishService.publish()` 在签发前校验数据集 `row_filter` 为空，否则拒绝 `is_public=true` |
| Notebook 调度状态卡 RUNNING | 30s 轮询 Dagster + 30min 超时兜底置 FAILED；UI 显示"超时未回写"标识 |
| Superset 数据源缺 tenant_id 导致 RLS 失效 | `SupersetEmbedService.verifyDatasourceHasTenantColumn()` 签发 guest token 前硬校验 |
| `notebook_run` 与历史 `JobRun` 的 RunStatus 命名漂移 | `module-common.RunStatus` enum 强校验；CI 跑 `RunStatusAuditTest` |
