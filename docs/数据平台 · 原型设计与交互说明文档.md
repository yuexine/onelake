## 4. 交互规则

1. 页面存在不等于业务完成。
2. 如果页面包含 mock/fallback，文档必须写“混合数据源”。
3. 危险操作必须使用确认和影响分析。
4. 长任务必须接入全局任务条或明确说明未接入。
5. 跨模块跳转必须说明携带参数和返回路径。
6. DWD 主体验应围绕业务治理流程；技术 DAG 作为高级视图。

## 5. 设计令牌

密级色和状态色继续沿用：

- L1 公开
- L2 内部
- L3 敏感
- L4 机密

涉及字段、资产、API 返回、脱敏策略、PII 扫描和 RBAC 时，必须使用统一 badge，不内联颜色。
