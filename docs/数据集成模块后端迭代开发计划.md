# 数据集成模块后端当前状态与后续迭代

> 修订日期：2026-07-09
>
> 本文由旧“迭代开发计划”收敛而来。历史计划中列为缺口的 discovery、Airbyte connection、运行日志、取消、reconcile、schema snapshot 等能力，当前代码已部分或主链路实现。

## 1. 当前代码事实

| 区域 | 当前实现 |
| --- | --- |
| Controller | `DataSourceController`、`SyncTaskController`、`CdcTaskController`、`FileSourceController`、`SourceSchemaSnapshotController`、`IntegrationMonitorController` |
| 数据源 | CRUD、测连、未保存配置测试、Airbyte connector definition/spec 查询、schema/table/column discovery |
| 采集任务 | 创建、列表、详情、更新、删除、启用、停用、dry-run、触发、运行详情、运行日志、取消、reconcile |
| Airbyte | source/destination/connection 准备、sync 触发、job snapshot、日志、取消、OAuth client credential 兼容 |
| schema | `SourceSchemaSnapshot`、drift 检测、`integration.schema.drift` 事件 |
| 事件 | `integration.sync_task.created`、`integration.table.loaded`、`integration.sync.failed`、`integration.sync_run.started` 等 |
| 前端 | 数据源、采集任务、向导、详情、运行日志主链路接 `IntegrationAPI`；CDC/模板/部分诊断仍混合 mock |

## 2. 当前 API 口径

统一前缀：`/api/v1/integration`

| 资源 | 代表接口 |
| --- | --- |
| 数据源 | `/datasources`、`/{id}/test`、`/test-config`、`/probe-databases` |
| Airbyte 元信息 | `/datasources/airbyte/source-definitions`、`/destination-definitions`、`/{definitionId}/spec` |
| 探查 | `/datasources/{id}/schemas`、`/tables`、`/tables/{objectName}/columns` |
| 采集任务 | `/sync-tasks`、`/{id}/enable`、`/{id}/disable`、`/dry-run`、`/{id}/run`、`/{id}/trigger` |
| 运行 | `/sync-tasks/runs/{runId}`、`/logs`、`/cancel`、`/reconcile` |
| CDC/文件 | `/cdc-tasks`、`/file-sources` |
| 监控 | `/monitor/health-summary`、`/monitor/fail-top` |

## 3. 主链路状态

```text
创建数据源
  -> 测连 / discovery
  -> 创建采集任务
  -> dry-run
  -> enable 时准备 Airbyte connection
  -> trigger 创建本地 sync_run 并触发 Airbyte job
  -> reconcile 回写状态、行数、checkpoint、错误
  -> 发布 integration.table.loaded / integration.sync.failed
  -> Catalog / Security / Orchestration / DataService 消费事件
```

历史联调记录显示 Postgres/Airbyte 主链路曾完成端到端验证；本轮文档修订没有重新启动数据面复测。

## 4. 剩余生产化项

| 优先级 | 项目 | 说明 |
| --- | --- | --- |
| P0 | Airbyte 运行配置显式化 | workspace、definition、destination/source config 缺失时给出更明确引导 |
| P0 | 调度闭环 | Dagster schedule reconcile 默认关闭，需完成生产配置和可观测反馈 |
| P0 | 运行日志体验 | Airbyte attempt/log 聚合、前端诊断、错误分类可继续增强 |
| P1 | CDC/file 生产化 | 当前有 Controller/Service 基础，真实 Flink/file collector 运行链路需继续联调 |
| P1 | 连接密钥治理 | 与 `module-security.secret` 的密钥托管和轮换还需补强 |
| P1 | E2E 自动化 | 为创建数据源 -> Airbyte -> Catalog/PII 增加可重复集成测试 |

## 5. 废弃口径

- `/api/integration` 旧前缀废弃，统一 `/api/v1/integration`。
- “缺少 update/delete/enable/disable/log/cancel/discovery”已不是当前事实。
- Airbyte 不在 Docker Compose 中启动。
