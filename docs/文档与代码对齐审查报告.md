# 文档与代码对齐审查报告

> 日期：2026-07-09
>
> 本次修订启动了 4 个只读子 Agent，分别审查后端、前端、数据面运行和文档叙事一致性；主 Agent 合并结论后完成全文口径收敛。

## 1. 审查结论

最大漂移不是某个单点错误，而是事实、验收、计划和原型混在同一层：

- 一些已完成的能力仍写成待开发。
- 一些 mock/原型页面被写成生产闭环完成。
- DWD 运行主线已从 dbt-on-Trino 转为 Spark pipeline，但多篇文档仍沿用旧口径。
- Airbyte 已改为 `abctl`，但部分文档仍按 Compose 或只写 `8000`。
- Flyway 范围是 10 schema，迁移目录是 9 个拥有表的 schema，旧“8/9 schema”说法需要移除。

## 2. 已统一口径

| 主题 | 新口径 |
| --- | --- |
| 后端模块 | 10 个 Maven 子模块；8 个业务模块 + common + bootstrap |
| 数据库 | 10 个 schema 范围；9 个迁移目录；约 73 张表 |
| DWD | Modeling 管定义，Orchestration 管 Spark-only 运行 |
| Airbyte | `abctl` 管理；`8000` 是 UI/ingress，`18001` 是后端 API fallback port-forward |
| 前端 | 分为已接 API、混合数据源、原型待接 |
| API 前缀 | 统一 `/api/v1/<module>` |
| 事件 | Outbox -> Redis Stream -> consumer group -> consumed_event |
| 状态 | 新主状态使用 `SUCCEEDED`，旧 `SUCCESS` 仅兼容 |

## 4. 后续维护规则

1. 改代码后先更新事实层，再更新模块附录。
2. 页面验收必须写数据来源，不把 mock 写成已完成。
3. 数据面验证必须分“容器进程、HTTP/CLI 探活、业务链路”。
4. DWD 文档不再新增 dbt-on-Trino 主线表述。
5. Airbyte 文档必须区分 UI、ingress、后端 API fallback 和 port-forward。
