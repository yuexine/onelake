# 本地开发环境完整部署指南

> 修订日期：2026-07-09
>
> 目标：明确 OneLake 本地控制面、前端、Compose 数据面和 Airbyte 子集群的启动边界。`make up` 只代表容器进程启动，不代表业务链路 ready。

## 1. 部署边界

| 层级 | 管理方式 | 说明 |
| --- | --- | --- |
| 后端控制面 | 本机 Maven 进程 | `bootstrap`，默认 `8080`，debug `5005` |
| 前端控制台 | 本机 pnpm/Vite | 默认 `5173` |
| Compose 数据面 | `docker compose -f docker-compose.yml` | Postgres、Redis、MinIO、Hive Metastore、Trino、Keycloak、OpenMetadata、PostgREST、APISIX、Dagster、Superset、JupyterHub、Spark、Flink、Kafka/Zookeeper、etcd |
| Airbyte | `abctl` + kind/Kubernetes | 由 `scripts/airbyte-local.sh` 管理，不在 Compose 中 |

## 2. 前置工具

| 工具 | 建议 |
| --- | --- |
| JDK | Java 17 |
| Maven | 3.9+ |
| Node/pnpm | Node 20 / pnpm 9+ |
| Docker Desktop | 至少 8 GB；同时跑 Airbyte 建议 12 GB+ |
| abctl | 仅 Airbyte 需要 |
| kubectl | 仅 Airbyte port-forward/排障需要 |

## 4. 后端启动

宿主机运行后端时，建议显式使用宿主机可达端点：

```bash
cd /Users/wuyuexin/CodeLab/onelake/onelake-app
export JDBC_URL=jdbc:postgresql://localhost:5432/onelake
export JDBC_USER=onelake
export JDBC_PASSWORD=onelake
export KEYCLOAK_URL=http://localhost:8081
export REDIS_HOST=localhost
export REDIS_PORT=6379
export MINIO_URL=http://localhost:9000
export MINIO_ACCESS_KEY=minio
export MINIO_SECRET_KEY=minio12345
export TRINO_JDBC_URL=jdbc:trino://localhost:18080/iceberg
export POSTGREST_URL=http://localhost:3001
export JUPYTERHUB_URL=http://localhost:18000
export KAFKA_BOOTSTRAP=localhost:9092

mvn -q -pl bootstrap -am install -DskipTests -Djacoco.skip=true
mvn -q -pl bootstrap spring-boot:run 2>&1 | tee ../.run-logs/backend.log
```

健康检查：

```bash
curl -fsS http://localhost:8080/actuator/health
curl -fsS http://localhost:8080/v3/api-docs >/dev/null
```

## 5. 前端启动

```bash
cd /Users/wuyuexin/CodeLab/onelake/onelake-app/web-console
pnpm install
pnpm dev --host 0.0.0.0 2>&1 | tee /Users/wuyuexin/CodeLab/onelake/.run-logs/web-console-vite.log
```

默认 `/api` 代理后端 `8080`。如需通过 APISIX：

```bash
VITE_API_PROXY_TARGET=http://localhost:9080 pnpm dev
```

注意：`make frontend` 当前会执行 `pnpm gen:api`，生成输出为 `src/api`，可能覆盖手写 API facade。执行前请确认生成目录策略。

## 6. Airbyte

Airbyte 不由 Compose 管理：

```bash
cd /Users/wuyuexin/CodeLab/onelake/onelake-app
AIRBYTE_LOW_RESOURCE_MODE=true make airbyte-up
make airbyte-status
make airbyte-credentials
```

Airbyte UI 默认：

```text
http://localhost:8000/
```

后端默认 Airbyte API：

```text
http://localhost:18001/api/v1
```

`18001` 只有启动 port-forward 后才可用：

```bash
kubectl --kubeconfig /Users/wuyuexin/.airbyte/abctl/abctl.kubeconfig \
  -n airbyte-abctl port-forward svc/airbyte-abctl-airbyte-server-svc 18001:8001
```

若 `8000` 返回 503 或 pod `CrashLoopBackOff`，先看：

```bash
kubectl --kubeconfig /Users/wuyuexin/.airbyte/abctl/abctl.kubeconfig get pods -A
abctl local status
```

## 8. 验证分层

### 8.1 容器进程

```bash
cd /Users/wuyuexin/CodeLab/onelake/onelake-app

```

Compose healthcheck 并不覆盖所有服务。Postgres、Redis、MinIO、OpenMetadata MySQL、Dagster PG、Dagster user-code 可看 health；Trino、Superset、JupyterHub 等应使用 HTTP/CLI 探活。

## 9. ODS -> DWD smoke 口径

`make ods-dwd-baseline` 和 `make ods-dwd-verify` 负责样例源表/ODS 基线，不等同于完整 DWD 运行闭环。完整验证应继续检查：

1. Dagster 暴露 `onelake_pipeline_run`。
2. 控制面/编排触发 Spark pipeline。
3. Dagster run 完成。
4. Trino 中 DWD 表可查询。
5. `job_run/task_run`、Catalog、Quality、Outbox 事件同步更新。

```bash
cd /Users/wuyuexin/CodeLab/onelake/onelake-app

```

`make airbyte-down` 会卸载本地 Airbyte 部署，可能删除 Airbyte 内部状态，仅在需要释放资源或重建时使用。
