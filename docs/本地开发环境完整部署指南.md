# 本地开发环境完整部署指南

> 最近验证日期：2026-06-29
>
> 适用目录：`/Users/wuyuexin/CodeLab/onelake`
>
> 目标：在本机同时拉起 OneLake 控制面、前端控制台、Compose 数据面组件，并明确 Airbyte 这个独立 `abctl/kind` 子集群的启动和排障边界。

## 1. 部署边界

| 层级 | 组件 | 管理方式 |
| --- | --- | --- |
| 控制面 | Spring Boot `bootstrap` | 本机 Maven 进程，默认 `8080` |
| 前端 | React/Vite `web-console` | 本机 pnpm/Vite 进程，默认 `5173` |
| Compose 数据面 | Postgres、Redis、MinIO、Hive Metastore、Trino、Keycloak、OpenMetadata、PostgREST、APISIX、Dagster、Superset、JupyterHub、Spark、Flink、Kafka、Zookeeper、etcd | `onelake-app/docker-compose.yml` |
| Airbyte | Airbyte + ingress-nginx + kind Kubernetes | `abctl`，通过 `onelake-app/scripts/airbyte-local.sh` 管理，不属于 Docker Compose |

Airbyte 使用宿主机 `8000`。因此 JupyterHub 在 Compose 中映射到 `18000:8000`，避免端口冲突。

### 3.2 初始化基础数据

```bash
make seed
make migrate
docker compose -f docker-compose.yml run --rm openmetadata ./bootstrap/openmetadata-ops.sh migrate
docker compose -f docker-compose.yml up -d openmetadata
```

说明：

- `make seed` 初始化 Keycloak realm 和 MinIO bucket。
- `make migrate` 按 schema 顺序执行 Flyway：`common`、`integration`、`orchestration`、`catalog`、`modeling`、`quality`、`security`、`dataservice`、`analytics`。
- OpenMetadata 首次启动前必须执行自身迁移，否则会报 `DATABASE_CHANGE_LOG doesn't exist` 或 `openmetadata_settings doesn't exist`。

## 4. Airbyte 独立启动

Airbyte 不由 Docker Compose 管理。需要采集链路时再启动：

```bash
cd /Users/wuyuexin/CodeLab/onelake/onelake-app
AIRBYTE_LOW_RESOURCE_MODE=true make airbyte-up
make airbyte-status
make airbyte-credentials
```

健康检查：

```bash
curl -i http://localhost:8000/api/v1/health
kubectl --kubeconfig /Users/wuyuexin/.airbyte/abctl/abctl.kubeconfig get pods -A
```

当前本地后端默认使用 `http://localhost:18001/api/v1` 访问 Airbyte API。若 `localhost:8000` 的 abctl ingress 出现 connection reset，可保留 Airbyte 子集群运行，并启动端口转发：

```bash
mkdir -p /Users/wuyuexin/CodeLab/onelake/.run-logs
screen -dmS onelake-airbyte-port-forward bash -lc \
  'kubectl --kubeconfig /Users/wuyuexin/.airbyte/abctl/abctl.kubeconfig -n airbyte-abctl port-forward svc/airbyte-abctl-airbyte-server-svc 18001:8001 > /Users/wuyuexin/CodeLab/onelake/.run-logs/airbyte-port-forward.log 2>&1'
curl -i http://localhost:18001/api/v1/health
```

如果 `http://localhost:8000/api/v1/health` 返回 `503`，并且 pods 出现 `CrashLoopBackOff`：

```bash
docker restart airbyte-abctl-control-plane

abctl local deployments
abctl local deployments --restart airbyte-abctl-server
abctl local deployments --restart airbyte-abctl-worker
abctl local deployments --restart airbyte-abctl-workload-api-server
abctl local deployments --restart airbyte-abctl-workload-launcher
abctl local deployments --restart airbyte-abctl-cron
```

若仍持续 TLS handshake timeout 或 CrashLoop，通常是 Docker Desktop 资源不足。可先停止不需要的重组件，或在确认可丢弃本地 Airbyte 状态后重建：

```bash
make airbyte-down
AIRBYTE_LOW_RESOURCE_MODE=true make airbyte-up
```

## 7. 常见问题

| 现象 | 原因 | 处理 |
| --- | --- | --- |
| `jupyterhub:4.1` 拉取失败 | 镜像缺少命名空间 | 使用 `jupyterhub/jupyterhub:4.1` |
| JupyterHub 和 Airbyte 端口冲突 | 二者容器内都用 `8000` | JupyterHub 宿主机端口固定为 `18000` |
| JupyterHub 启动报 `No module named oauthenticator` | 镜像未安装 OAuth 插件 | 重建 `jupyterhub` 镜像 |
| PostgREST 报 JWT secret 太短 | `PGRST_JWT_SECRET` 少于 32 字符 | 使用 `onelake-dev-secret-change-me-please-rotate` 或更长值 |
| APISIX 报 `control port 9180 conflicts with admin port` | control/admin 端口重叠 | control 端口使用 `9090` |
| APISIX 连接 `127.0.0.1:2379` 失败 | 新版 APISIX 读取 `deployment.etcd.host` | 将 etcd 配置放在 `deployment.etcd.host` |
| APISIX Admin 返回 `403` | 本地 admin 访问网段或 key 未配置 | 检查 `deployment.admin.allow_admin` 和 `X-API-KEY` |
| OpenMetadata 报 `DATABASE_CHANGE_LOG doesn't exist` | 未执行 OpenMetadata 自身迁移 | 跑 `docker compose run --rm openmetadata ./bootstrap/openmetadata-ops.sh migrate` |
| Airbyte 返回 `503` | kind 子集群 pod 未 ready | 查 `kubectl get pods -A`，按第 4 节重启；资源不足时使用 low resource mode |
