# 流水线与算子市场当前标准

> 修订日期：2026-07-09
>
> 旧文中“算子市场后端为 0”的表述已过期。当前已有算子、版本、安装、内置 seed、校验和前端市场入口。

## 1. 当前实现

| 层 | 当前代码 |
| --- | --- |
| 后端 | `OperatorController`、`OperatorService`、`BuiltInOperatorCatalog`、`OperatorSeeder` |
| 数据库 | `operator`、`operator_version`、`operator_install` |
| 前端 | `/orchestration/operators`、统一编辑器任务面板 |
| 事件 | pipeline publish/run/task loaded 相关事件常量 |

## 2. 当前算子口径

| 类型 | 当前定位 |
| --- | --- |
| `SYNC_REF` | 引用采集/ODS 输入 |
| `SPARK_SQL` | Spark SQL 转换 |
| `PYSPARK` | PySpark 脚本任务 |
| `QUALITY_GATE` | 质量门禁任务 |

## 3. 市场生命周期

当前支持或部分支持：

1. 内置算子 seed。
2. 算子列表和详情。
3. manifest 校验。
4. 注册/更新。
5. 版本管理。
6. 安装到租户/项目。

## 4. 后续标准化项

- manifest schema 固化并加版本。
- 算子输入/输出端口和字段 schema 校验增强。
- 私有算子、租户可见性、废弃/恢复策略生产化。
- 算子运行资源画像和安全沙箱。
- 前端对 `template/resourceHint` 的编辑体验增强。
