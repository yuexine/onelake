# ① 数据采集层（L1）

<aside>
🎯

**层目标**：以可视化、低代码方式，稳定地把多源异构数据（关系库/消息/日志/对象文件）按离线、实时、文件三种范式采集入湖（ODS），并保证一致性、可观测、可恢复。

### 🔧 L1-1 离线批采集 · 详细设计与技术实现

**技术选型**：抽取引擎 Spark 3.x DataSourceV2 + DataX 混合；调度 DolphinScheduler/Airflow（DAG）；口令托管 KMS；运行指标上报 Prometheus；元数据回写 Iceberg Catalog。

**L1-1.1 数据源连接管理 — 实现方案**

- **L1-1.1.1 连接创建**：`ConnectionService.create()` 先做 JSON Schema 校验（host 正则、port∈[1,65535]、type 枚举），口令采用 KMS 信封加密（`GenerateDataKey`→DEK 加密口令→落库存「密文+加密DEK」于 `auth_ref`，主密钥 CMK 不出 KMS）。落库后异步探活；幂等键 = `hash(tenant_id+type+host+port+db_name)` 防重复创建。
- **L1-1.1.2 连接测试**：按驱动下发 `SELECT 1` 或读取 `information_schema` 探查，超时 5s；失败按 网络(NET)/鉴权(AUTH)/驱动(DRV) 三类映射错误码并返回 RTT 时延。
- **L1-1.1.3 连接池配置**：基于 HikariCP，参数 `maxPoolSize / idleTimeout / connectionTimeout / validationQuery`；按源库规格设上限（默认 ≤ 源库 `max_connections` 的 20%），获取超时触发快速失败而非堆积。
- **L1-1.1.4 密钥轮换**：监听 KMS 轮换事件或定时触发，生成新 DEK 重加密 `auth_ref`，连接池执行「软重建」（新连接用新口令、旧连接 grace 期内自然回收），全程不停任务。
- **L1-1.1.5 连接权限隔离**：连接强绑 `tenant_id + project_id`，校验所用账号为只读（探测 `SHOW GRANTS`/`has_table_privilege`），具备写权限的账号直接拒绝创建。

**L1-1.2 抽取任务定义 — 实现方案**

- **L1-1.2.1 全量抽取**：以主键或数值列做 range/hash 分片（`WHERE pk BETWEEN ? AND ?`），分片并行度可配；每分片完成写 manifest（分片号、行数、min/max、校验和），全部成功后原子提交 ODS 快照。
- **L1-1.2.2 增量抽取(水位线)**：持久化 `high_watermark`，本次抽取区间 `(last_wm, now - safety_gap]`，`safety_gap` 默认回溯 N 分钟以容忍事务延迟；水位线单调递增，仅在分片全部成功后推进。
- **L1-1.2.3 字段映射**：维护源→目标类型映射矩阵（如 Oracle `NUMBER(p,s)`→`Decimal(p,s)`、`CLOB`→`String`），不兼容时按「可降级转换 / 告警 / 阻断」三档处理，映射规则随任务版本化。
- **L1-1.2.4 断点续传**：分片级 checkpoint 落 KV（`job_run_id + shard_id → status/offset`），失败仅重跑 `FAILED/RUNNING` 分片；重跑前清理该分片已写临时文件保证幂等。
- **L1-1.2.5 抽取限流**：令牌桶控制 QPS/带宽，并行度上限 + 错峰时间窗（cron 白名单）；源库负载（连接数/慢查询）超阈自动降速。
- **L1-1.2.6 采集前置脱敏**：在写 ODS 前的 transform 阶段调用 L4-3 脱敏算子，按字段密级路由算法，脱敏在内存流式完成、不落明文中间态。

**L1-1.3 调度与监控 — 实现方案**

- **L1-1.3.1 定时调度**：Cron + DAG 依赖触发；上游成功事件经消息总线驱动下游，支持「数据就绪信号」（分区 `_SUCCESS` 标记）触发。
- **L1-1.3.2 并发/隔离**：同任务用分布式锁（Redis/ZK）防重入；跨任务按资源队列隔离 CPU/内存配额，避免互相挤占。
- **L1-1.3.3 失败重试**：指数退避（base×2^n 带抖动）重试 N 次，区分可重试（网络抖动）与不可重试（语法/权限）错误，超限转告警并标记 `FAILED`。
- **L1-1.3.4 运行监控**：Exporter 上报行数/耗时/吞吐/失败率到 Prometheus，Grafana 看板 + Alertmanager 分级告警（钉钉/邮件/Webhook），行数突变用同环比阈值检测。

</aside>

### 🔧 L1-2 实时采集（CDC） · 详细设计与技术实现

**技术选型**：Flink CDC / Debezium 捕获，Kafka 作缓冲与解耦，Iceberg 作 sink，状态后端 RocksDB + 远端 checkpoint(S3)。

**L1-2.1 变更捕获 — 实现方案**

- **L1-2.1.1 Binlog/WAL 订阅**：MySQL 需 `binlog_format=ROW`、`binlog_row_image=FULL`，账号授 `REPLICATION SLAVE/CLIENT`；PG 用逻辑复制槽（`wal_level=logical` + replication slot），捕获 I/U/D 并保留 before/after 镜像。
- **L1-2.1.2 初始快照+增量衔接**：先锁定起始位点(GTID/LSN)再做全量快照，快照期间增量进 Kafka 暂存，快照完成后从记录位点回放增量无缝衔接，保证不丢不重。
- **L1-2.1.3 位点管理**：offset/LSN 随 Flink checkpoint 原子持久化，故障重启从最近成功 checkpoint 恢复；位点提交与 sink 提交两阶段对齐。
- **L1-2.1.4 Exactly-Once**：source 端 checkpoint + sink 端 Iceberg 两阶段提交（pre-commit 写数据文件、commit 提交快照），配合主键幂等 upsert 实现端到端一次语义。

**L1-2.2 Schema 演进感知 — 实现方案**

- **L1-2.2.1 DDL 捕获**：解析 Debezium schema change 事件，将 ADD/DROP/MODIFY COLUMN 转为 Iceberg `ALTER TABLE` 演进动作。
- **L1-2.2.2 兼容性策略**：加列/类型提升等向后兼容变更自动执行；删列/改窄类型等破坏性变更进入审批队列，未审批前缓冲数据按旧 schema 写入。
- **L1-2.2.3 演进通知**：变更写入元数据中心并经血缘触发下游影响通知（订阅 Webhook / 站内信）。

| 编号 | 叶子功能 | 详细说明 | 优先级 |
| --- | --- | --- | --- |
| L1-3.1.1 | 目录监听 | 轮询/事件方式监听源目录(FTP/SFTP/NAS/对象桶)新增文件 | P0 |
| L1-3.1.2 | 大文件分片上传 | 分片并行上传 S3 + 断点续传 + 合并；支持 GB 级文件 | P0 |
| L1-3.1.3 | MD5/SHA 校验 | 上传前后校验一致性，不一致重传并告警 | P0 |
| L1-3.1.4 | 文件元数据登记 | 登记文件名、大小、类型、来源、哈希、落地路径至元数据中心 | P0 |

### 🔧 L1-3 文件采集 · 详细设计与技术实现

**技术选型**：监听基于 inotify / 对象存储事件通知(S3 Event→SQS)，传输用 S3 Multipart Upload，校验用 MD5/SHA-256，元数据回写 Catalog。

- **L1-3.1.1 目录监听**：SFTP/NAS 用定时轮询 + 文件稳定性判定（大小连续两次不变才视为写完）；对象桶优先用事件通知（S3→SNS/SQS）即时触发，避免轮询延迟与漏采。
- **L1-3.1.2 大文件分片上传**：按 part（默认 64/128MB）切分并行 `UploadPart`，记录已传 part 实现断点续传，全部完成后 `CompleteMultipartUpload` 合并；单文件分片数 ≤ 10000。
- **L1-3.1.3 MD5/SHA 校验**：上传前算本地摘要，上传后用 ETag / 对象摘要比对，不一致自动重传并告警；分片场景按 part ETag 聚合校验。
- **L1-3.1.4 文件元数据登记**：登记文件名/大小/类型/来源/哈希/落地路径/采集时间至元数据中心，作为血缘起点与去重依据。
- **L1-3.1.5 去重**：以内容哈希为唯一键，命中已登记哈希则跳过入湖并记一条引用关系，避免重复占用存储。

# ② 湖仓存储层（L2）

<aside>
🎯

**层目标**：基于 S3 + 开放表格式(Iceberg/Hudi)构建分层(ODS/DWD/DWS/ADS)、具备 ACID/时间旅行/Schema 演进的湖仓底座，并提供存储优化与统一 SQL 查询。

</aside>

## L2-1 分层建模

| 编号 | 叶子功能 | 详细说明 | 优先级 |
| --- | --- | --- | --- |
| L2-1.1.1 | 四层定义 | ODS(贴源)/DWD(明细)/DWS(汇总)/ADS(应用)，每层物理库前缀隔离 | P0 |
| L2-1.1.2 | 命名规范校验 | 建表强制 {层}*{域}*{业务}_{粒度} 规范，违规拦截 | P1 |
| L2-1.1.3 | 生命周期(TTL) | 按层配置保留周期，过期分区自动归档/清理 | P1 |
| L2-1.1.4 | 分层依赖校验 | 禁止跨层逆向依赖(如 ODS 读 ADS)，建模时校验 | P2 |

### 🔧 L2-1 分层建模 · 详细设计与技术实现

**技术选型**：Iceberg 表 + REST Catalog；分层以 namespace/库前缀隔离；建模元数据入入元数据中心；TTL/归档由调度作业驱动。

- **L2-1.1.1 四层定义**：ODS/DWD/DWS/ADS 各占独立 namespace（如 `ods_*`/`dwd_*`），物理路径也按层分桶隔离，便于权限与生命周期差异化管理。
- **L2-1.1.2 命名规范校验**：建表时用正则校验 `{层}_{域}_{业务}_{粒度}` 格式，违规在 DDL 网关拦截并返回修改建议；域/粒度枚举取自元数据字典。
- **L2-1.1.3 生命周期(TTL)**：按层配置保留天数，定时作业扫描过期分区→先转归档桶再删除元数据，删除前检查血缘下游是否仍引用。
- **L2-1.1.4 分层依赖校验**：建模/发布时解析任务血缘，禁止逆向依赖（如 ODS 读 ADS），违规阻断并提示重构依赖路径。

## L2-2 表格式管理

| 编号 | 叶子功能 | 详细说明 | 优先级 |
| --- | --- | --- | --- |
| L2-2.1.1 | ACID 事务 | 基于 Iceberg 快照实现原子提交，并发写冲突检测重试 | P0 |
| L2-2.1.2 | Schema 演进 | 列增删改、重命名、类型提升，不重写历史数据 | P0 |
| L2-2.1.3 | 时间旅行 | 按 snapshot-id / 时间戳查询历史版本，支持回滚 | P1 |
| L2-2.1.4 | Upsert/Merge | 基于主键 MERGE INTO，支撑 CDC 入湖去重更新 | P0 |
| L2-2.1.5 | 快照管理 | 过期快照清理(expire_snapshots)、孤儿文件清理 | P1 |

### 🔧 L2-2 表格式管理 · 详细设计与技术实现

**技术选型**：Apache Iceberg（snapshot / manifest / data file 三层元数据），MERGE-ON-READ + COPY-ON-WRITE 混合策略。

- **L2-2.1.1 ACID 事务**：基于 Iceberg 快照的原子提交（原子替换 metadata 指针）；并发写用乐观锁，提交冲突时根据基线快照重试。
- **L2-2.1.2 Schema 演进**：基于列 ID（非名称）跟踪，支持加/删/改名/类型提升且不重写历史数据，读取时按快照 schema 映射。
- **L2-2.1.3 时间旅行**：按 `snapshot-id` 或时间戳读历史版本（`FOR SYSTEM_TIME AS OF`），支持 `rollback_to_snapshot` 回滚。
- **L2-2.1.4 Upsert/Merge**：以主键 `MERGE INTO ... WHEN MATCHED/NOT MATCHED`，CDC 入湖用 equality-delete + position-delete 去重更新。
- **L2-2.1.5 快照管理**：定时 `expire_snapshots`（保留 N 个/N 天） + `remove_orphan_files` 清理孤儿文件，避免 metadata 膨胀。

## L2-3 存储优化

| 编号 | 叶子功能 | 详细说明 | 优先级 |
| --- | --- | --- | --- |
| L2-3.1.1 | 分区策略 | 按时间/业务键分区，支持 Iceberg 隐藏分区避免分区谓词错误 | P0 |
| L2-3.1.2 | 小文件合并 | 定时 Compaction，合并小文件到目标大小(128/256MB) | P0 |
| L2-3.1.3 | Z-Order/排序 | 多维聚簇排序，提升多列过滤裁剪率 | P2 |
| L2-3.1.4 | 冷热分层 | 访问频率低的分区下沉低频/归档存储桶降本 | P1 |
| L2-3.1.5 | 列存格式 | Parquet/ORC + 压缩(ZSTD/Snappy)，统计信息收集 | P0 |

### 🔧 L2-3 存储优化 · 详细设计与技术实现

**技术选型**：Spark/Trino 维护作业 + Iceberg 表服务（`rewrite_data_files`/`rewrite_manifests`）；冷热分层对接对象存储 storage-class。

- **L2-3.1.1 分区策略**：优先用 Iceberg 隐藏分区（`days(ts)`/`bucket(n,key)`），查询无需显式写分区谓词，避免谓词错误导致全表扫描。
- **L2-3.1.2 小文件合并**：定时/按小文件数阈值触发 Compaction，将小文件合并至目标大小（128/256MB），并行度可控避免影响写入。
- **L2-3.1.3 Z-Order/排序**：多维聚簇排序（Z-Order/Hilbert）提升多列过滤的文件裁剪率，适用于高频多条件查询表。
- **L2-3.1.4 冷热分层**：按访问热度将低频分区下沉低频/归档 storage-class（如 S3 IA/Glacier），访问时透明回调或预热。
- **L2-3.1.5 列存格式**：Parquet/ORC + ZSTD/Snappy 压缩，写入时收集列统计（min/max/null count）供谓词下推与 CBO。

## L2-4 查询引擎

| 编号 | 叶子功能 | 详细说明 | 优先级 |
| --- | --- | --- | --- |
| L2-4.1.1 | 统一 SQL 网关 | 统一 JDBC/SQL 入口，路由到 Trino(交互)/Spark(批) | P0 |
| L2-4.1.2 | 资源队列 | 按租户/优先级分配资源组，防止大查询挤占 | P1 |
| L2-4.1.3 | 结果缓存 | 相同查询结果缓存，加速重复访问 | P2 |
| L2-4.1.4 | 查询审计 | 记录 SQL、执行人、扫描量、耗时，供成本与安全分析 | P0 |

### 🔧 L2-4 查询引擎 · 详细设计与技术实现

**技术选型**：Trino（交互/联邦） + Spark（批/ETL），统一 SQL 网关层路由，资源组隔离。

- **L2-4.1.1 统一 SQL 网关**：提供统一 JDBC/SQL 入口，按查询类型（交互低延迟→Trino；大批量→Spark）路由，屏蔽引擎差异。
- **L2-4.1.2 资源队列**：按租户/优先级划分 Trino resource group，限制单查询 CPU/内存/并发，防大查询挤占集群。
- **L2-4.1.3 结果缓存**：按 SQL 指纹 + 表快照版本缓存结果，底表发生新提交时自动失效，保证一致性。
- **L2-4.1.4 查询审计**：记录 SQL、执行人、扫描量、耗时、路由引擎，供成本分摊与安全审计（异常扫描告警）。

<aside>
⚠️

</aside>

| 编号 | 叶子功能 | 详细说明 | 优先级 |

### 🔧 L4-1 数据治理 · 详细设计与技术实现

**技术选型**：算子跑在 Spark/Flink，码表/规则集中配置并版本化，与 L3-4 质量门禁联动。

| 编号 | 叶子功能 | 详细说明 | 优先级 |

### 🔧 L4-2 数据加密 · 详细设计与技术实现

**技术选型**：TLS 终端 + S3 SSE-KMS + 应用层字段加密，密钥全部托管 KMS，信封加密。

- **L4-2.1.1 传输加密**：全链路强制 TLS1.2+（禁用弱套件），证书由 ACME/内部 CA 统一签发与自动续期，到期告警。
- **L4-2.1.2 存储加密**：S3 桶级强制 SSE-KMS 服务端加密（bucket policy 拒绝未加密上传），按租户隔离 CMK。

**技术选型**：脱敏算子库可插拔，静态脱敏跑批任务，动态脱敏在查询/网关层拦截改写结果。

- **L4-3.1.1 静态脱敏**：落库前对副本永久脱敏，生成可用于测试/分析环境的脱敏数据集，不可逆。
- **L4-3.1.2 动态脱敏**：查询/API 返回时按调用方角色实时重写字段（改写 SQL 投影或结果拦截），原数据不变。
- **L4-3.1.3 脱敏算法**：掩码、替换、泛化、哈希、加密、随机化，按字段类型配置（如手机号保留前3后4）。
- **L4-3.1.4 策略联动分级**：按 L3-5.1.2 密级自动应用对应脱敏策略，免逐字段配置，密级变更时策略随动。
- **L4-3.1.5 一致性脱敏**：同一原值脱敏后结果一致（确定性哈希/加盐映射），保证跨表 join 关联不断。

## L4-4 处理编排

| 编号 | 叶子功能 | 详细说明 | 优先级 |
| --- | --- | --- | --- |
| L4-4.1.1 | DAG 拖拽编排 | 可视化拖拽算子连线，自动校验环路/类型匹配 | P0 |
| L4-4.1.2 | 算子市场 | 内置 + 自定义算子注册，输入输出schema声明 | P1 |
| L4-4.1.3 | 参数化 | 全局/环境变量、运行时参数，支持多环境复用 | P1 |
| L4-4.1.4 | 版本管理 | 流水线版本化、对比、回滚、灰度发布 | P1 |
| L4-4.1.5 | 调试与试运行 | 采样数据预览每个算子输出，所见即所得 | P1 |

### 🔧 L4-4 处理编排 · 详细设计与技术实现

**技术选型**：前端 DAG 画布生成 pipeline DSL，运行时编译为 Spark/Flink 作业，算子以 schema 声明接口插拔。

- **L4-4.1.1 DAG 拖拽编排**：可视化拖拽算子连线，保存前自动校验环路（拓扑排序）与上下游 schema 类型匹配。
- **L4-4.1.2 算子市场**：内置 + 自定义算子注册，声明输入/输出 schema 与参数，版本化发布，支持租户私有算子。
- **L4-4.1.3 参数化**：全局/环境变量 + 运行时参数（如 `${run_date}`），支持多环境（dev/test/prod）复用同一流水线。
- **L4-4.1.4 版本管理**：流水线版本化、版本 diff、一键回滚、灰度发布（按比例或指定环境先行）。
- **L4-4.1.5 调试与试运行**：采样数据预览每个算子输出（所见即所得），试运行不落正式表、仅写临时区。

<aside>
⚠️

**异常与边界**：算子失败→失败策略(中断/跳过/重试)可配；脱敏与加密同字段冲突→以最高安全策略优先；DAG 环路→保存前拦截；密钥不可用→处理任务挂起不降级输出明文。

</aside>

# ⑤ 数据服务层 / DaaS（L5）

<aside>
🎯

**层目标**：以低代码方式把 ADS 数据快速封装为安全、可计量、可治理的对外 API，统一网关接入，结合鉴权/限流/动态脱敏对外开放。

</aside>

## L5-1 API 构建

| 编号 | 叶子功能 | 详细说明 | 优先级 |
| --- | --- | --- | --- |
| L5-1.1.1 | SQL 建 API | 编写参数化 SQL(:param) 自动生成 REST 接口 | P0 |
| L5-1.1.2 | 向导式建 API | 选表→选字段→配过滤→生成，无需 SQL | P1 |
| L5-1.1.3 | 参数映射与校验 | 入参类型/必填/范围校验，防 SQL 注入(预编译) | P0 |
| L5-1.1.4 | 结果裁剪/分页 | 字段白名单、分页(limit/offset或游标)、排序 | P0 |
| L5-1.1.5 | API 测试与文档 | 在线调试 + 自动生成 OpenAPI 文档 | P1 |

### 🔧 L5-1 API 构建 · 详细设计与技术实现

**技术选型**：参数化 SQL 模板引擎 + 预编译（PreparedStatement），后端路由到 Trino，OpenAPI 自动生成。

- **L5-1.1.1 SQL 建 API**：编写参数化 SQL（`:param`）自动生成 REST 接口，引擎解析占位符生成入参元数据，绑定数据源与资源组。
- **L5-1.1.2 向导式建 API**：选表→选字段→配过滤条件→生成，后台自动拼接安全 SQL，无需手写，适合业务人员。
- **L5-1.1.3 参数映射与校验**：入参类型/必填/范围校验，全部走预编译防 SQL 注入，非法参数返回 400 并记审计。
- **L5-1.1.4 结果裁剪/分页**：字段白名单输出，分页支持 limit/offset 与游标（大结果集强制游标），可指定排序列。
- **L5-1.1.5 API 测试与文档**：在线调试面板 + 自动生成 OpenAPI/Swagger 文档，变更时文档同步更新。

## L5-2 API 网关

| 编号 | 叶子功能 | 详细说明 | 优先级 |
| --- | --- | --- | --- |
| L5-2.1.1 | 路由转发 | 统一域名 + 路径路由到后端数据服务 | P0 |
| L5-2.1.2 | 版本管理 | /v1 /v2 多版本并存，废弃版本灰度下线 | P1 |
| L5-2.1.3 | 灰度发布 | 按比例/按消费方灰度新版本，可快速回退 | P2 |
| L5-2.1.4 | 协议转换 | 统一对外 REST，对内可转 gRPC | P2 |

### 🔧 L5-2 API 网关 · 详细设计与技术实现

**技术选型**：基于 APISIX/Kong 或自研网关，插件化架构（路由/鉴权/限流/转换为可插拔插件）。

- **L5-2.1.1 路由转发**：统一域名 + 路径路由到后端数据服务，支持按 header/参数的动态路由与负载均衡。
- **L5-2.1.2 版本管理**：`/v1` `/v2` 多版本并存，废弃版本标记 Deprecation 头并灰度下线，到期后返回 410。
- **L5-2.1.3 灰度发布**：按比例或按消费方（AppKey 白名单）灰度新版本，异常可一键回退到稳定版。
- **L5-2.1.4 协议转换**：对外统一 REST，对内可转 gRPC，网关处理序列化/反序列化与超时传递。

## L5-3 安全管控

| 编号 | 叶子功能 | 详细说明 | 优先级 |
| --- | --- | --- | --- |
| L5-3.1.1 | 鉴权 | OAuth2/AppKey+Secret/JWT，按 API 粒度授权 | P0 |
| L5-3.1.2 | 限流熔断 | 按消费方/API 配 QPS/并发限流，后端异常熔断降级 | P0 |
| L5-3.1.3 | IP 白名单 | 限制可调用来源 IP/网段 | P1 |
| L5-3.1.4 | 动态脱敏返回 | 结合 L4-3.1.2 按消费方角色脱敏返回字段 | P0 |
| L5-3.1.5 | 越权防护 | 行级/列级权限校验，防止水平/垂直越权 | P0 |

### 🔧 L5-3 安全管控 · 详细设计与技术实现

- **L5-3.1.1 鉴权**：支持 OAuth2 / AppKey+Secret / JWT，按 API 粒度授权；AppKey 签名验证 + 时间戳防重放，Token 短期 + refresh。
- **L5-3.1.2 限流熔断**：按消费方/API 配 QPS/并发限流（滑动窗口/令牌桶），后端错率超阈熔断降级，半开探测恢复。
- **L5-3.1.3 IP 白名单**：限制可调用来源 IP/网段，支持按消费方配置，命中黑名单直接拒绝并告警。
- **L5-3.1.4 动态脱敏返回**：结合 L4-3.1.2，网关按消费方角色对返回字段实时脱敏，策略与密级联动。
- **L5-3.1.5 越权防护**：行级/列级权限校验（注入租户/权限过滤条件），防水平（跨租户）与垂直（越级）越权。

- **L5-4.1.1 调用日志**：记录调用方、参数、耗时、返回码、扫描量，异步写日志库/ES 可检索，不阻塞主链路。
