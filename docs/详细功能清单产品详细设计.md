# OneLake 产品功能清单与当前实现映射

> 修订日期：2026-07-09
>
> 本文从原 PDD 详细清单收敛为“产品目标 + 当前实现映射”。旧文中的 `/api/v1/collect`、`/api/v1/daas` 和大量未来式技术方案不再作为当前代码事实。当前接口统一使用 `/api/v1/integration`、`/api/v1/dataservice` 等模块前缀。

## 1. 状态定义

| 状态 | 含义 |
| --- | --- |
| 已实现 | 后端和/或前端已有真实实现 |
| 混合 | 真实 API 与原型/mock 并存 |
| 原型 | 前端可见或设计明确，但后端闭环未完成 |
| 目标 | 产品目标，当前不应写成已交付 |

## 2. 功能层映射

| 层 | 模块 | 当前状态 | 当前代码入口 |
| --- | --- | --- | --- |
| L1 数据采集 | 数据源、采集任务、运行监控、CDC、文件源 | 已实现 + 混合 | `module-integration`、`/integration/*` |
| L2 湖仓建模 | 表管理、SQL、DWD、存储优化 | 已实现 + 混合 | `module-catalog`、`module-modeling`、`module-orchestration`、`/lakehouse/*` |
| L3 治理 | 目录、血缘、质量、术语、资产安全 | 已实现 + 混合 | `module-catalog`、`module-quality`、`module-modeling`、`module-security` |
| L4 开发编排 | Spark pipeline、算子市场、运行实例 | 已实现 | `module-orchestration`、`/orchestration/*` |
| L5 数据服务 | API 草稿、发布、调试、订阅、网关 | 混合 | `module-dataservice`、`/dataservice/*` |
| L6 分析可视化 | 数据集、Dashboard、Notebook、分享、NL2SQL | 已实现但依赖外部服务验证 | `module-analytics`、`/analytics/*` |
| 平台运营 | 监控、告警、系统管理 | 原型 + 局部 API | `module-common`、`module-security`、前端 `/monitor/*`、`/system/*` |

## 3. 当前核心用户旅程

### 3.1 数据接入到目录可见

```text
创建数据源 -> 探查 schema/table/columns -> 创建采集任务
-> Airbyte 同步 -> reconcile -> Catalog 资产/字段/血缘 -> PII 标签
```

状态：主链路已实现；Airbyte 环境和配置需单独验收。

### 3.2 ODS 到 DWD

```text
ODS 资产 -> DWD 模型定义/校验/编译 -> Spark pipeline
-> DWD 表 -> Catalog 血缘 -> Quality 结果
```

状态：当前以 Spark-only pipeline 为主线；dbt-on-Trino 是历史方案。

### 3.3 SQL 到 API 草稿

```text
SQL Workbench -> 保存查询/执行 -> 发布为 API 草稿
-> API 构建向导 -> 调试 -> 发布/下线
```

状态：草稿、调试和部分发布链路已实现；完整外部 Trino-backed API 网关仍需生产化。

### 3.4 分析大屏分享

```text
数据集 -> Dashboard -> 发布 -> 公开分享 -> 分享页访问
```

状态：代码已实现；Superset/JupyterHub/OpenAI 等外部依赖需单独验证。

## 5. 后续产品化重点

1. 把 Monitor/System 原型接入真实后端。
2. 把 DataService AppKey/订阅/网关页面从 mock 完成到真实闭环。
3. 增强 CDC/file collector 真实运行链路。
4. 血缘图去 fallback，完成大图交互和性能。
5. 统一全局任务、通知、告警投影。
6. 为 Airbyte、Spark pipeline、Analytics 外部依赖补 E2E 自动化。
