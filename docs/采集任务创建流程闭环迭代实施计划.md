# 采集任务创建流程闭环当前状态

> 修订日期：2026-07-09
>
> 本文保留为采集任务闭环附录。旧文中“尚未闭合”的表述已与当前代码不一致，现统一为“主 Postgres/Airbyte 链路已具备代码闭环，生产化与自动化验证继续推进”。

## 1. 当前闭环

| 步骤 | 当前状态 |
| --- | --- |
| 选择数据源 | 前端向导从真实 `IntegrationAPI.listDataSources` 加载 |
| schema/table/column 探查 | 后端 MySQL/Postgres discovery，向导失败时显式错误 |
| 字段映射 | 基于真实 columns 生成，避免样例字段兜底 |
| 保存草稿 | `POST /api/v1/integration/sync-tasks` |
| 发布/启用 | `POST /{id}/enable`，准备 Airbyte connection |
| 试跑 | `dry-run` 做配置与资源前置检查 |
| 触发运行 | `/{id}/trigger` 先落本地 run，再触发 Airbyte |
| 运行回写 | `/runs/{runId}/reconcile` 写状态、行数、checkpoint、error |
| 日志/取消 | `/logs`、`/cancel` 已有后端和前端入口 |
| 下游事件 | 成功发 `integration.table.loaded`，失败发 `integration.sync.failed` |

## 2. 关键事件

| 事件 | 当前用途 |
| --- | --- |
| `integration.sync_task.created` | Security 触发 PII 扫描，DataService 预生成草稿 |
| `integration.table.loaded` | Catalog 建档/刷新字段与血缘，Orchestration 触发依赖 |
| `integration.sync.failed` | Catalog/告警记录失败 |
| `integration.schema.drift` | 标记结构漂移，等待下游治理 |

## 3. 当前仍需收口

1. Airbyte workspace/definition/destination/source 配置缺失时，产品引导还不够清晰。
2. Dagster 调度登记默认关闭，生产环境需配置 `DAGSTER_SCHEDULE_ENABLED=true` 并验证 reconcile job。
3. CDC 与文件采集有控制面入口，但真实运行链路需要继续联调。
4. Airbyte 2.x API、OAuth、attempt 统计兼容需要持续用集成测试覆盖。
5. 历史已联调记录不是本轮复测结果；环境变更时必须重新跑业务 smoke。

## 4. 验证建议

```bash
cd onelake-app
mvn -q -pl module-integration -am test -Djacoco.skip=true
```

完整链路需额外启动 Airbyte、后端、前端和源库，验证：

```text
创建数据源 -> 探查 -> 创建任务 -> 发布 -> 触发 -> reconcile
-> 目标库查数 -> Catalog 资产 -> PII 标签/血缘
```
